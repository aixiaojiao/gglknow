GglKnow 推文收藏器 帮助文档

🚀 快速上手

  1. 点击浏览器工具栏的插件图标，打开设置弹窗。
  2. **务必先设置“保存路径”**，这是所有收藏内容存放的位置。
  3. 在 Twitter/X 网站上，找到推文下方的“收藏”或“收藏串”按钮。
  4. 点击即可将推文（或整个推文串）完整保存到你设置的路径中。

的核心功能

1. 收藏单个推文 / 推文文章V2.1.5 新增

在任意普通推文或长篇推文文章下方，点击蓝色的 **“收藏”** 按钮，即可将当前推文的完整内容、图片和视频保存下来。

2. 收藏推文串 (Thread)

当一条推文是某个推文串的一部分时，你会看到一个额外的 **“收藏串”** 按钮。点击它，扩展会自动遍历并保存整个串的所有推文。

3. 推文浏览器

一个内置的本地工具，用于浏览和管理你所有收藏过的推文。在插件弹窗中点击 **“打开推文浏览器”** 即可进入。功能包括：

  • 内容浏览: 集中展示所有推文，格式与 Twitter 类似。
  • 搜索与筛选: 支持按关键词、作者、时间范围进行筛选和排序。
  • 图片恢复: 由于浏览器安全限制，图片可能无法自动加载。点击 **“🔄 Reload Images”** 按钮并重新选择你当初设置的保存文件夹，即可恢复所有图片显示。

⚙️ 设置详解

  • 保存路径：设置文件保存的文件夹。
    路径小贴士：
    - **Windows 用户**: 可以使用 D:\Twitter-Saves 这样的绝对路径，或者 Downloads\Tweets 这样的相对路径（相对于浏览器默认下载目录）。
    - **macOS / Linux 用户**: 可以使用 /Users/YourName/Twitter 这样的绝对路径。
  • 文件格式：可多选，满足不同需求：
    • HTML: 最佳选择，生成带样式的网页，保留原始排版，浏览体验最好。
    • JSON: 包含所有原始数据，适合数据备份或二次开发。
    • Markdown: 纯文本格式，方便导入到 Notion、Obsidian 等笔记软件。
  • 下载媒体文件：默认开启，会自动下载推文中的所有图片和视频。

常见问题 (FAQ)

问：为什么点击“收藏”后没有反应？

答：最常见的原因是 **没有设置“保存路径”**。请打开插件设置，确保该项已填写。

问：更新扩展后，收藏按钮点击无效怎么办？

答：这是因为扩展更新后上下文失效了。请 **刷新一下 Twitter/X 的网页**，按钮就会恢复正常。

问：推文浏览器里的图片不显示，怎么办？

答：这是正常的浏览器安全机制。请点击浏览器顶部的 **“🔄 Reload Images”** 按钮，在弹出的窗口中选择你设置的保存文件夹，所有图片就会立刻显示出来。

问：收藏下来的推文文章格式错乱？

答：我们已在 V2.1.5 版本中对文章格式进行了大幅优化。请确保你的扩展已更新到最新版。如果问题依旧，欢迎通过 GitHub 向我们报告。
